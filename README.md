# 3D Галактика – визуализация молекулярного излучения

Интерактивное трёхмерное приложение для изучения распределения молекулярного излучения (CO, OH, H₂O, HCO⁺) в нашей Галактике.  
Вы можете вращать и масштабировать модель, включать/отключать отображение разных молекул и получать детальные характеристики любого объекта по клику.

## О проекте

Проект создан в рамках исследовательской работы **«Пространственное распределение молекулярного состава Галактики согласно данным наблюдений»** (паспорт №21/ЛКП-735-2026 от 03.02.2026).  
Цель – дать учёным и всем интересующимся инструмент для наглядного анализа положения и яркости молекулярных облаков в галактической системе координат.

Визуализация построена на реальных астрономических данных: для каждого объекта известны галактические координаты (l, b), расстояние от Солнца (r), лучевая скорость (v_r), интенсивности излучения по разным молекулам и квантовые переходы.

---

## Функциональность

- **Трёхмерная модель Галактики** – 500 000 точек, моделирующих распределение звёзд (балдж, диск, гало, спиральные рукава).  
- **Объекты молекулярного излучения** – цветные сферы, размер и яркость которых зависят от интенсивности излучения выбранных молекул.  
- **Фильтрация по молекулам** – можно показывать только CO, OH, H₂O, HCO⁺ или любые комбинации.  
- **Интерактивное управление** – вращение, панорамирование, масштабирование (мышь + клавиатура).  
- **Информация по клику** – при нажатии на сферу внизу экрана появляются:
  - название объекта
  - галактические координаты l, b
  - расстояние r (пк)
  - лучевая скорость v_r (км/с)
  - суммарная интенсивность
  - квантовые переходы (например, `J=1-0`, `J=3-2`)

---

## Требования

- **Браузер** с поддержкой WebGL (Google Chrome, Mozilla Firefox, Microsoft Edge, Safari).  
- **Локальный веб-сервер** – файлы нельзя открывать просто двойным щелчком (из-за политики CORS).  
- **Python 3** (только если нужно пересобрать `merged_objects.json` из Excel).  
- **Оперативная память** – от 4 ГБ (для 500 000 точек).  
- **Видеокарта** – любая с поддержкой WebGL.

## Установка и запуск

### 1. Скачайте проект

Склонируйте репозиторий или скачайте ZIP-архив и распакуйте в любую папку.

### 2. Запустите локальный веб-сервер

Так как приложение загружает файл `merged_objects.json`, его нужно отдавать через HTTP. Проще всего использовать встроенный сервер Python:

```bash
# Откройте терминал в папке проекта
cd /путь/к/папке/с/index.html

# Для Python 3
python -m http.server 8080

Если у вас установлен Node.js, можно использовать live-server:

```bash
npx live-server
```

Или любой другой сервер (например, MAMP, Open Server, расширение Live Server для VS Code).

### 3. Откройте приложение

В браузере перейдите по адресу:  
`http://localhost:8080` (или другому порту, который вы указали).

Через несколько секунд появится трёхмерная сцена.

> ⚠️ Если вы видите только чёрный экран – проверьте, что файл `merged_objects.json` загрузился (смотрите вкладку Network в инструментах разработчика).

## Работа с приложением

### Управление камерой

| Действие | Мышь | Клавиатура |
|----------|------|------------|
| Вращение | Зажать левую кнопку и двигать | – |
| Панорамирование | Зажать правую кнопку и двигать | W, A, S, D (или стрелки) |
| Приближение / отдаление | Колёсико мыши | – |

### Фильтрация молекул

В левом нижнем углу находятся переключатели для CO, OH, H₂O, HCO⁺.

- **Включён** – сферы, имеющие эту молекулу с ненулевой интенсивностью, будут видны.  
- **Выключен** – сферы, у которых нет активных молекул, скрываются.

Цвет каждой сферы формируется как взвешенная смесь цветов активных молекул (вес = интенсивность).  
Яркость свечения пропорциональна максимальной интенсивности среди активных молекул.

### Получение информации об объекте

Щёлкните левой кнопкой мыши по любой цветной сфере.  
В нижней центральной панели (`data-container`) появятся все характеристики.

Если кликнуть на пустое место, информация очистится, а подсветка объекта пропадёт.

## Формат входных данных

Все данные хранятся в файле `merged_objects.json`.  
Пример одного объекта:

```json
{
  "Name": "Sgr B2 (Main)",
  "l": 0.667,
  "b": -0.035,
  "r": 8200.0,
  "v_r": 62.0,
  "Transitions": {
    "HCO+": [ "J=1-0" ]
  },
  "CO": null,
  "H2O": null,
  "OH": null,
  "HCO+": 4.5
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| Name | строка | Название источника |
| l | число | Галактическая долгота (градусы) |
| b | число | Галактическая широта (градусы) |
| r | число | Расстояние от Солнца в парсеках (пк) |
| v_r | число | Лучевая скорость (км/с) |
| Transitions | объект | Ключи – молекулы (CO, OH, H2O, HCO+), значения – массивы строк с переходами |
| CO, OH, H2O, HCO+ | число или null | Интенсивность излучения для данной молекулы |

> **Примечание:** поле `Transitions` используется только для отображения в панели информации. Оно может содержать переходы даже для молекул, у которых интенсивность `null`.

## Генерация merged_objects.json

Исходные данные могут поставляться в виде четырёх Excel-файлов:  
`CO.xlsx`, `OH.xlsx`, `H2O.xlsx`, `HCO+.xlsx`.

Файлы должны иметь одинаковую структуру: колонки `Name`, `l`, `b`, `r`, `v_r`, `Intensity`, `Transition`.

Скрипт `Data_analyze.py` объединяет их в единый `merged_objects.json`:

- Читает каждый Excel.
- Заменяет запятые на точки в числах (для европейского формата).
- Группирует записи по `Name`, собирая переходы в объект `Transitions`.
- Сохраняет результат в JSON.

**Запуск скрипта**
```bash
python Data_analyze.py
```

Убедитесь, что установлены библиотеки `pandas` и `openpyxl`:
```bash
pip install pandas openpyxl
```

После выполнения появится файл `merged_objects.json`, который сразу можно использовать в приложении.

## Настройка (конфигурация)

### Параметры звёздного фона

Файл `src/config.js`:
```javascript
export const Total_Points = 500000;       // количество звёзд
export const R_Disk = 50000;              // радиус диска (пк)
export const R_Buldge = 3260;             // радиус балджа (пк)
export const R_Halo = 100000;             // радиус гало (пк)
export const Fraction_Buldge = 0.1;       // доля звёзд в балдже
export const Fraction_Disk = 0.8;         // доля звёзд в диске
export const Fraction_Galo = 0.1;         // доля звёзд в гало
```

### Внешний вид сфер

- Базовый радиус сферы задаётся в `main.js` при вызове `createObjectMaterials(data, 350)`.
- Цвета молекул – в `src/materials/objectMaterials.js` (константа `CRITERIA_COLORS`).
- Яркость свечения регулируется в шейдере (`createGlowSphereMaterial`, параметр `uIntensity`).

### Масштаб координат

В функции `astroToCartesian` (файл `objectMaterials.js`) используется множитель `scale = 3.2`, чтобы объекты не сливались с Солнцем (`R_Sun = 26080`). При необходимости измените этот коэффициент.

## Возможные проблемы и решения

### 1. Не загружается `merged_objects.json`

**Причина:** политика CORS при открытии `index.html` напрямую.  
**Решение:** запустите локальный веб-сервер (см. раздел «Установка и запуск»).

### 2. Сферы слишком большие или маленькие

**Решение:** измените `baseRadius` (в `main.js`) и/или масштаб в `astroToCartesian`.

### 3. Тормозит при вращении

**Решение:** уменьшите `Total_Points` в `config.js` (например, до 200 000). Также можно отключить эффект Bloom (закомментировать `composer` в `main.js`).

### 4. Некоторые объекты не отображаются

Проверьте:

- Включены ли соответствующие фильтры (CO, OH …).
- Есть ли у объекта ненулевая интенсивность для активных молекул (значение > 0).
- Не «улетели» ли координаты за пределы видимости – попробуйте приблизить/отдалить камеру.

### 5. Excel-файлы не читаются в `Data_analyze.py`

**Решение:** убедитесь, что установлены `pandas` и `openpyxl`. Если в файлах используются десятичные запятые, скрипт их корректно заменяет на точки.
